# Stanford University Code in Place: Clinical Systems Portfolio 🩺💻
**Engineered by TUYISHIMIRE Emmanuel**

This repository contains a suite of high-fidelity, Python-based healthcare applications developed during the rigorous **Stanford University Code in Place (2026)** engineering program. 

Bridging the gap between clinical nursing protocols and software engineering, these applications demonstrate the practical deployment of complex Python fundamentals—including advanced conditional logic, data validation, and graphical coordinate mapping—to solve real-world healthcare administration and patient triage challenges.

---

## 1. FirstLine Clinical TriageOS 
**Type:** Terminal-based Clinical Decision Support System (CDSS)
**Core Tech:** Advanced Conditional Logic, `try/except` Data Validation, `datetime` Integration.

Designed to address bottlenecks and data-entry errors in Emergency Department intakes. The engine securely ingests patient vitals, utilizing continuous validation loops to prevent impossible medical values from entering the system. TriageOS cross-references these inputs against standard physiological baselines to automatically assign patient acuity (Critical, Urgent, or Routine). To mirror enterprise EMRs, the system instantly generates unique patient tracking IDs, exact timestamps, and outputs actionable, evidence-based clinical next steps.

## 2. WardFlow: Clinical Resource Visualizer
**Type:** Spatial Dashboard & Real-Time Resource Engine
**Core Tech:** Stanford `graphics` Canvas Library, `for` Loops, Coordinate Array Mapping, `random` libraries.

A graphical dashboard engineered to simulate real-time hospital ward management. Built utilizing Stanford's specialized Canvas library, the application dynamically generates a hospital wing architecture and simulates live patient occupancy. The dashboard relies on complex X/Y coordinate mapping to plot the exact physical locations of critical mobile supplies (e.g., Crash Carts and Medication Carts) relative to the central Nurses' Station, optimizing spatial awareness for clinical staff.

## 3. HepB Health Advocate Engine
**Type:** Public Health Interactive Console
**Core Tech:** Python Dictionaries, Dynamic Scoring Loops, User Input Handling.

An interactive, logic-driven educational tool designed to raise clinical awareness regarding Hepatitis B transmission and preventative care. The engine utilizes dynamic looping to quiz users, processing their inputs to generate real-time scoring and deliver targeted clinical facts based on performance metrics.
